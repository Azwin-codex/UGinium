# UGinium Color & Material Token System v1.0

## Introduction

The UGinium Color System is built around materials rather than colors.

Users should perceive the interface as a precision-engineered environment made from graphite, titanium, and subtle energy.

Color exists only to communicate identity, interaction, and meaning.

---

# Color Philosophy

Every color must answer one question.

"What information does this color communicate?"

No color exists purely for decoration.

---

The visual hierarchy is divided into four layers.

Material Layer

Only these tokens change.

# Layer 3 — Interaction Colors

These colors communicate system state.

Never change background color dramatically.

Pressed

Focus

Loading

Disabled

Lower opacity.

Reduced contrast.

---

# Border Rules

Borders are structural.

Never decorative.

Use Titanium at low opacity.

Increase opacity only during interaction.

Borders should define architecture rather than decoration.

---

# Shadow Rules

Shadows are subtle.

Use ambient depth.

Avoid dramatic floating effects.

Elevation should be communicated through spacing before shadows.

---

# Glow Rules

Glow is reserved for interaction.

Never leave glowing elements idle.

Allowed

Orb

Hover

Focus

XP

Signals

Notifications

Forbidden

Entire cards

Large backgrounds

Continuous neon lighting

---

Everything else remains identical.

↓

Primary Accent

Derived Orange

↓

VIT Campus

Primary Accent

Derived Blue

↓

IIT Campus

Primary Accent

Derived Red

The Academy remains visually consistent.

---

# Accessibility

All Campus colors must pass WCAG AA contrast.

Accent colors should never reduce readability.

Text always takes priority over branding.

---

# Final Rule

Materials create trust.

Color creates identity.

Motion creates emotion.

Never allow color to overpower the architecture.

The Academy is remembered for its structure.

Color merely tells the user where they belong.
