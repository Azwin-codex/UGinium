# UGinium Typography & Information System v1.0

## Introduction

Typography is the voice of the Academy.

It does not simply display information.

It establishes hierarchy, communicates authority, and reinforces progression.

Every word should feel deliberate.

Every number should feel meaningful.

Typography must remain calm, premium, and engineered.

---

# Typography Philosophy

UGinium typography follows four principles.

Clarity

Precision

Hierarchy

Restraint

Typography should never compete with architecture.

It complements it.

---

# Letter Spacing

Display Titles

Slightly tighter

Headings

Neutral

Labels

Slightly wider

Noticeably wider

Example

LEVEL 42

MISSION STATUS

ACADEMY CORE

Small increases in tracking create precision.

# Numeric Hierarchy

Numbers are extremely important in UGinium.

Every important number should immediately stand out.

Examples

Level

XP

Guild Rank

Project Count

Achievement Count

---

# Level System

The Level is the strongest visual element on every profile.

Hierarchy

LEVEL

↓

Number

↓

Class

↓

Student Name

Never allow the avatar to dominate the Level.

Progression defines identity.

---

# XP Display

XP should always appear in monospaced typography.

Example

2840 / 3500 XP

The alignment reinforces precision.

XP bars should always accompany numerical values.

Never show XP visually without numbers.

---

# Module Titles

Every module begins with a structural title.

Examples

IDENTITY

MISSIONS

ARCHIVE

SIGNALS

PROJECTS

ACHIEVEMENTS

These titles should be subtle.

They orient the user.

They should never overpower the content.

---

# Labels

Labels communicate structure.

Examples

Guild

University

Availability

Class

Role

Status

Use uppercase.

Increase letter spacing.

Reduce font size.

Labels should feel like technical interface markings.

---

# Paragraph Rules

Paragraphs should remain short.

Maximum width

Approximately 70 characters.

Avoid large walls of text.

Interfaces are scanned.

Not read like books.

---

# Alignment

Numbers

Right aligned where comparison matters.

Text

Left aligned.

Titles

Left aligned.

Statistics

Grid aligned.

Never center long text.

---

# Reading Rhythm

Every module should follow a predictable rhythm.

Title

↓

Primary Information

↓

Supporting Information

↓

Metadata

↓

Actions

The eye should naturally move downward.

---

# Icons & Typography

Icons support text.

They never replace labels.

Maintain consistent spacing.

Icons should align to the text baseline.

---

# Button Typography

Buttons should feel confident.

Avoid

CLICK HERE

JOIN NOW!!!

Instead

Join Guild

View Profile

Start Mission

Explore Academy

Button text should sound calm.

Not promotional.

---

# Empty States

Empty states should encourage exploration.

Examples

No Missions Yet

Discover your first mission.

No Guild

Find teammates and build something extraordinary.

No Projects

Every legacy begins with one project.

The Academy should always invite action.

---

# Recognition Rule

If all colors are removed,

the interface should remain unmistakably UGinium.

Typography alone should communicate

Precision

Progression

Academy

Structure

This is the ultimate test.

---

# Final Principle

Architecture builds the interface.

Typography gives it intelligence.

Numbers give it purpose.

Every word, every digit, and every label should reinforce one feeling:

"I am inside the Academy."
